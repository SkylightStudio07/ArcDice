## 1. 게임 개요

> 주사위 굴려 보드를 돌며 깃발을 모으는 로컬 핫시트 2인 보드배틀.  
> 100% Orange Juice / 마리오 파티에서 영감.

| 항목 | 내용 |
|---|---|
| 장르 | 보드배틀 / 파티게임 |
| 엔진 | Unity (2D) |
| 플랫폼 | PC (키보드 로컬 2인) |
| 세션 길이 | 1판 10~15분 목표 |
| 레퍼런스 | 100% Orange Juice (보드/전투), 마리오 파티 (이벤트 노드) |
| 개발 형태 | 동아리 게임잼 (18:00 ~ 익일 09:00) → 데모 우선, 이후 확장 |

### 핵심 재미 포인트
- **이동 vs 교전의 선택**: 앞서가는 플레이어는 도망치고, 뒤처진 플레이어는 전투로 따라잡는다
- **운과 전략의 긴장**: 주사위 기반이지만 아이템 사용 타이밍이 결과를 바꾼다
- **역전 가능성**: 전투 승리 시 깃발 강탈 = 끝까지 긴장감 유지

## 2. 게임 구조

### 2.1 승리 조건
- **깃발 3개 선취**

### 2.2 깃발 획득 방법

| 방법 | 조건 |
|---|---|
| 한 바퀴 완주 | 보드를 한 바퀴 돌아 시작 칸에 복귀 시 깃발 +1 |
| 전투 승리 (vs 플레이어) | 상대의 깃발 1개 강탈 |

### 2.3 턴 흐름

1. **턴 시작**: 현재 플레이어 표시, 아이템 확인 가능
2. **주사위 굴림**: 1~6 중 하나
3. **이동**: 굴린 수만큼 전진
4. **도착 칸 효과 처리**
   - 같은 칸에 상대 플레이어 있음 → **PvP 전투**
   - 배틀 노드 → **AI 중립몹 전투**
   - 이벤트 노드 (부스트/함정/워프) → 효과 발동
   - 일반 노드 → 아무 일 없음
5. **한 바퀴 체크**: 시작 칸 통과 시 깃발 +1
6. **승리 판정**: 깃발 3개면 종료, 아니면 다음 플레이어 턴

## 3. 보드 & 노드

### 3.1 보드 구조
- 단일 루프형 보드, **20칸 내외**
- **아이소메트릭 뷰 (45도 기울임)** — 2D 스프라이트 시각적 기울임만 (진짜 Isometric Tilemap 미사용)
- 칸 간 이동은 인덱스 기반 (`List<Transform>`)

### 3.2 노드 유형

| 노드 | 색상 가안 | 효과 | 데모 분포 |
|---|---|---|---|
| 일반 | 흰색 | 아무 일 없음 | 6~8칸 |
| 부스트 | 초록 | 주사위 1회 추가 굴림 | 3~4칸 |
| 함정 | 빨강 | 1턴 쉼 | 2~3칸 |
| 워프 | 보라 | 랜덤 노드로 순간이동 | 2~3칸 |
| 배틀 | 주황 | AI 중립몹과 전투. 승리 시 아이템 획득 | 3~4칸 |

## 4. 전투 시스템

### 4.1 공통 규칙

| 항목 | 수치 |
|---|---|
| HP | 4 |
| 주사위 | 1d6 (공격/방어/회피 모두 동일) |

### 4.2 전투 흐름

1. **전투 개시 UI**: 양측 플레이어가 아이템 사용 여부 선택
2. **선공 결정**: 양쪽 1d6 굴림, 높은 쪽 선공 (동률 시 재굴림)
3. **공격 페이즈** (공격자): 1d6 굴림 → 결과 = 데미지
4. **방어 페이즈** (방어자): 방어 / 회피 중 선택
   - **방어**: 1d6 굴림, 데미지 - 방어값 (최소 1 보장)
   - **회피**: 1d6 굴림, 공격값 이상이면 0뎀 / 미만이면 풀뎀
5. **HP 체크**: 0 이하면 전투 종료, 아니면 공수 교대 후 반복

### 4.3 전투 결과

| 전투 유형 | 승자 보상 | 패자 페널티 |
|---|---|---|
| vs 플레이어 (PvP) | 상대 깃발 1개 강탈 | 깃발 1개 상실 (0개면 없음) |
| vs AI 중립몹 | 아이템 1개 획득 | HP는 다음 전투 시작 시 회복 (데모: 매 전투 리셋) |

> 전투 간 HP — 데모에서는 **매 전투 리셋**으로 단순화. 추후 런 구조 도입 시 재검토.

## 5. 아이템 시스템

### 5.1 획득
- 배틀 노드 승리 보상
- 이벤트 노드 (추후 추가 가능)

### 5.2 사용 규칙
- **전투 개시 직전** UI에서 사용 여부 결정
- 1전투당 1개만 사용 가능 (가안)
- 보유 상한: **3개** (가안)

### 5.3 아이템 목록 (초안)

| 이름 | 효과 |
|---|---|
| 고정 주사위 | 다음 주사위 굴림 결과가 4/5/6 중 하나로 고정 |
| 공격 버프 | 이번 전투 동안 공격 결과 +2 |
| 방어 버프 | 이번 전투 동안 방어 결과 +2 |
| 도주 | 전투 회피. 1칸 뒤로 물러남 (깃발 강탈 방지) |

## 6. UI & 조작

### 6.1 로컬 핫시트 조작

| 기능 | P1 | P2 |
|---|---|---|
| 확인/선택 | Space | Enter |
| 취소/아이템 UI | Left Shift | Right Shift |

### 6.2 주요 UI 화면
- **메인 보드 화면**: 보드 + 말 + HUD (현재 턴, 각 플레이어 깃발/HP/아이템)
- **주사위 롤 애니메이션**: 숫자 텍스트 깜빡 연출 (3D 굴림 NO)
- **전투 개시 UI**: 아이템 사용 여부 선택
- **전투 화면**: 별도 패널 오버레이 or 별도 씬
- **승리 화면**: 결과 표시 + 재시작

## 8. 기술 설계

| 데이터 | 방식 | 비고 |
|---|---|---|
| 보드 칸 | `List<Transform>` + 인덱스 | Tilemap 미사용 |
| 노드 유형 | ScriptableObject or enum | 효과 함수 매핑 |
| 아이템 | ScriptableObject | 이름, 설명, 효과 타입 |
| 전투 | 별도 패널 or 씬 | 보드 상태 static hold |
| 턴 관리 | GameManager (싱글톤) | 상태머신 기반 |
